# Автоматизация тестирования веб-сервиса Excursium.com

Данный проект содержит набор автоматизированных UI-тестов для проверки функциональности авторизации, регистрации и восстановления пароля на сайте `excursium.com`.

## 🛠 Стек технологий
* **Python 3.x** — основной язык разработки.
* **Selenium WebDriver** — инструмент для управления браузером.
* **Pytest** — фреймворк для организации и запуска тестов.
* **Page Object Model (POM)** — паттерн проектирования для разделения логики тестов и описания страниц.

## 📋 Покрытие тестами (Test Suite)

Проект включает в себя **14 тестовых сценариев**, разделенных по функциональным блокам:

### 🔐 Авторизация (Login)
* **Позитивные:** Вход с валидными данными (Email/Password).
* **Негативные:** Проверки с пустыми полями, неверным форматом Email, коротким и невалидным паролем (7 сценариев).

### 📝 Регистрация (Registration)
* **Позитивные:** Регистрация с валидными данными и проверка появления формы подтверждения (кнопка "Подтвердить").
* **Граничные значения:** Проверка регистрации с паролем в 5 и 6 символов.
* **Негативные:** Регистрация дубликата пользователя, ввод слишком коротких паролей (1 и 4 символа).

### 🔄 Навигация и восстановление
* Проверка переключения между вкладками "Вход" и "Регистрация".
* Проверка перехода на форму восстановления пароля и появления кнопки "Отправить код".
